## 1. クイックスタート（最短で動かす）

### 1-1. ファイルを作る

```python
# scripts/sites/jobs/my_site.py

import sys
from pathlib import Path
root_path = Path(__file__).resolve().parent.parent.parent.parent
sys.path.insert(0, str(root_path))

from src.framework.static import StaticCrawler
from src.const.schema import Schema

class MySiteScraper(StaticCrawler):
    DELAY = 1.0  # リクエスト間隔（秒）

    def parse(self, url):
        soup = self.get_soup(url)
        for item in soup.select(".company"):
            yield {
                Schema.NAME: item.select_one(".name").text,
                Schema.TEL:  item.select_one(".tel").text,
            }

if __name__ == "__main__":
    import logging
    logging.basicConfig(level=logging.INFO)
    MySiteScraper().execute("https://example.com")
```

### 1-2. 実行する

```bash
python scripts/sites/jobs/my_site.py
# → output/20260303_MySiteScraper.csv が生成される
```

**これだけ。** CSV 保存、正規化、ログ、エラーハンドリングは全自動。

## 2. 開発者が知るべき 3 つのこと

### ① 実装するのは `parse()` メソッドだけ

```python
def parse(self, url):
    # データを dict で yield → フレームワークが全部やる
    yield { Schema.NAME: "テスト株式会社", Schema.TEL: "03-1234-5678" }
```

### ② カラム名は `Schema` 定数を使う

```python
from src.const.schema import Schema

Schema.NAME       # "名称"
Schema.ADDR       # "住所"
Schema.TEL        # "TEL"（全角→半角は自動変換）
Schema.HP         # "HP"
Schema.URL        # "取得URL"
# ... 全19カラム（→ 付録A参照）
```

Schema にないカラムを使いたい場合は `EXTRA_COLUMNS` で宣言する:

```python
class MyScraper(StaticCrawler):
    EXTRA_COLUMNS = ["企業概要", "設立年"]
```

### ③ `StaticCrawler` か `DynamicCrawler` を選ぶ

| 基底クラス | ライブラリ | 用途 |
|-----------|-----------|------|
| `StaticCrawler` | Requests + BeautifulSoup | 通常の HTML サイト |
| `DynamicCrawler` | Playwright (ヘッドレスブラウザ) | JS レンダリングが必要なサイト |

## 3. フレームワーク全体の処理フロー

```
execute(url) が呼ばれると…

  ① _setup()              ← セッション / ブラウザの起動（自動）
  ② prepare()             ← [Hook] 前処理（ログイン等、任意）
  ③ parse(url)            ← ★ 開発者が実装 ★
  │   └─ yield {data}
  │       └─ pipeline.process_item()
  │           ├─ バリデーション（未宣言カラム → ValueError）
  │           ├─ 正規化（TEL全角→半角, None→空文字）
  │           └─ CSV 書き込み
  ④ finalize()            ← [Hook] 後処理（任意）
  ⑤ pipeline.close()      ← CSV 確定・一時ファイル削除
  ⑥ _teardown_resources() ← セッション / ブラウザの解放（自動）
```

## 4. クラス変数リファレンス

```python
class MyScraper(StaticCrawler):
    DELAY = 1.0              # リクエスト間隔（秒）。デフォルト 0.0
    EXTRA_COLUMNS = ["備考"]  # 独自カラム。デフォルト []
```

| 変数 | 型 | デフォルト | 説明 |
|------|-----|----------|------|
| `DELAY` | `float` | `0.0` | `parse()` 内の各 yield 後の待機秒数 |
| `EXTRA_COLUMNS` | `list[str]` | `[]` | Schema 以外に使用するカラム名 |
| `CONTINUE_ON_ERROR` | `bool` | `True` | エラー耐性モード。`True` でエラー時スキップ継続、`False` で即停止 |

## 5. 便利メソッド・プロパティ

### StaticCrawler 専用

| メソッド | 説明 |
|---------|------|
| `self.get_soup(url)` | URL を取得して `BeautifulSoup` を返す（リトライ3回付き）。`CONTINUE_ON_ERROR=True` 時、通信エラーで `None` を返す |
| `self.session` | `requests.Session` オブジェクト（直接使用も可） |

### DynamicCrawler 専用

| メソッド / プロパティ | 説明 |
|-----------|------|
| `self.get_soup(url)` | Playwright でページを取得して `BeautifulSoup` を返す。`CONTINUE_ON_ERROR=True` 時、エラーで `None` を返す |
| `self.page` | Playwright の `Page` オブジェクト |
| `self.browser` | Playwright の `Browser` オブジェクト |
| `self.context` | Playwright の `BrowserContext` |

### 共通

| プロパティ | 説明 |
|-----------|------|
| `self.logger` | クラス名付きロガー |
| `self.total_items` | 設定すると ETA（残り時間）が自動表示される |
| `self.output_filepath` | 実行後の CSV ファイルパス |
| `self.observed_columns` | 実行中に観測されたカラム名リスト |
| `self.item_count` | 取得件数 |
| `self.error_count` | スキップされたエラー件数（`CONTINUE_ON_ERROR=True` 時） |
| `self.site_name` | CSV ファイル名に使うサイト名（日本語OK） |

## 6. Hook メソッド（任意オーバーライド）

### `prepare()` — 前処理

ログインや初期設定が必要な場合:

```python
def prepare(self):
    # 例: ログインが必要なサイト
    self.session.post("https://example.com/login", data={
        "username": "user", "password": "pass"
    })
```

### `finalize()` — 後処理

後片付けやレポート出力が必要な場合:

```python
def finalize(self):
    self.logger.info("処理完了！ %d件取得しました", self.item_count)
```

## 7. 進捗表示（ETA）

`self.total_items` に全件数をセットするだけで ETA が自動表示される:

```python
def parse(self, url):
    urls = self._collect_urls(url)
    self.total_items = len(urls)  # ← この1行だけ

    for u in urls:
        yield self._scrape(u)
```

**出力例:**

```
[50/200件] 残り約 00:07:30 (2.5件/秒)
```

未設定時は速度だけ表示:

```
[50件] 経過 00:02:30
```

**全件数の計算方法:**

- 方法1: URL リストの `len()` を使う
- 方法2: サイトに「全328件」と書いてあればパースする
- 方法3: 47都道府県 × 推定20件/県 のように概算する

## 8. エラーハンドリング

### エラー耐性モード (`CONTINUE_ON_ERROR`)

デフォルトで有効 (`True`)。エラー発生時にクローラー全体を停止させず、スキップして継続します。

```python
class MyScraper(StaticCrawler):
    CONTINUE_ON_ERROR = False  # エラー時に即停止させたい場合
```

### フレームワークが自動で処理するエラー一覧

| エラー種別 | CONTINUE_ON_ERROR=True (デフォルト) | CONTINUE_ON_ERROR=False |
|-----------|------|------|
| HTTP クライアントエラー (404, 403 等) | `get_soup()` が None を返す。スキップ | 例外を発生させて停止 |
| HTTP サーバーエラー (500, 502, 503, 504) | 自動リトライ 3 回 → 失敗なら None | 自動リトライ 3 回 → 失敗なら例外 |
| 接続エラー (タイムアウト、DNS等) | `get_soup()` が None を返す。スキップ | 例外を発生させて停止 |
| Playwright エラー (ページ読込失敗等) | `get_soup()` が None を返す。スキップ | 例外を発生させて停止 |
| 個別アイテムの処理失敗 | スキップして次へ（ログに Warning） | 例外を発生させて停止 |
| Schema 違反（未宣言カラム） | `ValueError` で即座に停止 | `ValueError` で即座に停止 |
| 正規化エラー（フォーマット不正等） | 値を空文字(`""`)として保存 | 値を空文字(`""`)として保存 |
| `KeyboardInterrupt` (Ctrl+C) | 即座に停止 | 即座に停止 |

スキップされたエラー件数は完了ログ・Teams 通知・Prefect ダッシュボードで確認できます。

### `parse()` 内でのエラーハンドリング推奨パターン

`CONTINUE_ON_ERROR=True` の場合、`get_soup()` は通信エラー時に `None` を返します。None チェックを入れると安全です:

```python
def parse(self, url):
    for detail_url in detail_urls:
        soup = self.get_soup(detail_url)
        if soup is None:
            continue  # エラーは自動ログ済み、次へ
        yield self._extract_data(soup)
```

独自のエラー処理を追加したい場合:

```python
def parse(self, url):
    for detail_url in detail_urls:
        try:
            yield self._scrape_detail(detail_url)
        except Exception as e:
            self.logger.warning("スキップ: %s (%s)", detail_url, e)
            continue  # 1件失敗しても次へ
```

### 統括層（Flow）でのエラー対応と通知

クローラー全体がクラッシュするような致命的なエラーが発生した場合、`bin/run_flow.py` にて以下の処理が自動で行われます:

1. **ステータス更新**: DB (`data_catalog` テーブル) の `last_run_status` が `failed` になります。
2. **Teams 通知**: `.env` で `TEAMS_WEBHOOK_URL` が設定されている場合、エラー内容と経過時間が Microsoft Teams に通知されます。


```bash
docker compose exec worker python bin/deploy.py
```
